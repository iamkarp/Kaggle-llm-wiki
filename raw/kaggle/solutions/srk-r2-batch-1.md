## 1. IEEE-CIS Fraud Detection (2019) — 1st Place (Part 2 Write-up)

**Task type:** Binary classification (transaction fraud detection)
**Discussion:** [https://www.kaggle.com/c/ieee-fraud-detection/discussion/111308](https://www.kaggle.com/c/ieee-fraud-detection/discussion/111308)

**Approach:** The FraudSquad team built an ensemble of XGBoost, CatBoost, and LightGBM models on top of heavily engineered features. The defining insight was constructing a synthetic "unique client ID" (UID) from card1 + addr1 + D1, then deriving group-aggregated features from that UID to encode per-client behavioral context across transactions.

**Key Techniques:**
1. **Client UID Construction** — Combined card1, addr1, and time-difference feature D1 to create a unique credit card client identifier. All subsequent aggregation features were computed at the UID level, drastically improving signal strength.
2. **Group Aggregation Feature Engineering** — Generated mean, std, count, and nunique aggregations of existing columns grouped by the UID and other categorical groupings (device type, email domain). These were the single largest LB jump features.
3. **Target / Frequency Encoding** — Applied target encoding and frequency encoding to high-cardinality categoricals (card types, device info, email domain), with proper out-of-fold splitting to avoid leakage.
4. **Gradient Boosting Ensemble** — Final submission blended LGBM, XGBoost, and CatBoost predictions. Post-processing replaced per-transaction predictions with the client-level average prediction within each UID group.
5. **UID-based Post-processing** — After scoring, transactions sharing the same UID had their fraud probabilities averaged, exploiting the fact that a client is either a fraudster or not uniformly.

**How to Reuse:**
- In any transaction/event dataset, invest early in building a synthetic entity key (user, card, device) and derive statistical aggregations at that entity level — this almost always yields the biggest feature gains.
- For fraud/anomaly tasks, post-process predictions at the entity level (average or max across all rows for an entity) to enforce consistency.
- Ensemble diverse GBM flavors (LGBM speed + XGBoost regularization + CatBoost categorical handling) for a reliable 1-3% gain over any single model.

## 2. Santander Customer Transaction Prediction (2019) — 2nd Place

**Task type:** Binary classification (customer will make a specific transaction)
**Discussion:** [https://www.kaggle.com/c/santander-customer-transaction-prediction/discussion/88939](https://www.kaggle.com/c/santander-customer-transaction-prediction/discussion/88939)

**Approach:** KazukiOnodera's solution used LightGBM with a novel feature construction strategy: treating each of the 200 anonymized features independently. The key discovery was that the test set contained synthetic (fake) samples generated by the competition organizers, which could be detected and removed. Features were count-encoded at multiple rounding precisions, then each feature was treated as its own model input in a "feature bundle" ensemble.

**Key Techniques:**
1. **Synthetic Sample Detection and Removal** — Identified fake test samples by finding feature values that appeared only once in the test set (a statistical fingerprint of synthetic data). Removing these before training provided a cleaner signal and avoided fitting to noise.
2. **Multi-Precision Count Encoding** — For each of 200 features, created four count-encoded variants by rounding to 0, 1, 2, and 3 decimal places. This captured value-frequency information at multiple granularities without feature interactions.
3. **Feature-Independent Modeling** — Trained a separate LightGBM model for each "5-feature bundle" (raw + 4 count encodings), stacking 195 × 10-fold = 1,950 models. This treats features as independent signals — unusual but highly effective for this dataset.
4. **Odds Multiplication Ensemble** — Combined predictions using probabilistic odds multiplication rather than simple averaging, weighting each model by its discriminative power.
5. **10-Fold Stratified CV** — Used stratified k-fold with k=10 to maximize data usage on this relatively small dataset and produce reliable OOF estimates for feature selection (kept only features with AUC ≥ 0.500).

**How to Reuse:**
- Always probe competition test sets for synthetic/augmented samples — value uniqueness patterns often reveal them, and removing them can dramatically improve CV-to-LB alignment.
- For datasets with many independent anonymized features, try per-feature model ensembles rather than training a single joint model; sometimes features truly are independent and joint modeling hurts.
- Count encoding at multiple decimal-rounding levels is a low-effort, high-signal trick for anonymized continuous features in classification tasks.

## 3. Deepfake Detection Challenge (2020) — 1st Place

**Task type:** Binary video classification (real vs. deepfake)
**Discussion:** [https://www.kaggle.com/c/deepfake-detection-challenge/discussion/145721](https://www.kaggle.com/c/deepfake-detection-challenge/discussion/145721)

**Approach:** Selim Seferbekov's winning solution used a frame-by-frame classification approach rather than complex video-level temporal modeling. Five EfficientNet-B7 models (trained with different random seeds on ImageNet + Noisy Student weights) processed individual video frames after MTCNN face detection and cropping. A "confident prediction" heuristic aggregated frame-level scores into a video-level prediction.

**Key Techniques:**
1. **Frame-Level EfficientNet-B7 Ensemble** — Extracted 32 frames per video and classified each independently with five B7 models trained on different seeds. Video-level score was aggregated from frame scores, avoiding the complexity of 3D CNNs or RNNs.
2. **Dynamic Face Detection and Cropping** — Used MTCNN with resolution-aware scaling (2x scaling for small videos <300px, 0.5x for large >1000px) and added 30% margin around face crops to include context artifacts introduced by deepfake generation.
3. **Confident Prediction Heuristic** — Instead of plain mean: if >11 frames scored >0.8 fake AND represented >40% of frames, averaged only high-confidence frames; if >90% of frames scored <0.2, averaged low-confidence values; otherwise simple mean. This reduced noise from frames with partial faces.
4. **Heavy Augmentation Pipeline** — Applied image compression artifacts, noise injection, geometric transforms, and GridMask-inspired dropout augmentations during training to simulate realistic video compression conditions seen in test data.
5. **Synchronized Batch Normalization + Separate Fake/Real Loss** — Computed fake loss and real loss separately within each mini-batch across 4 GPUs with SyncBN, preventing class imbalance from collapsing gradients toward the majority class.

**How to Reuse:**
- For video classification, frame-level image models often outperform complex temporal architectures — try the simple baseline first before adding 3D CNNs or LSTMs.
- Dynamic resolution scaling in face/object detection is critical: always adjust detector sensitivity based on input image size rather than using fixed thresholds.
- The "confident prediction" aggregation pattern — selectively averaging only high- or low-confidence predictions based on consensus — is broadly applicable to any multi-instance bag prediction problem.

## 4. Deepfake Detection Challenge (2020) — Public 2nd / Private 30th Place

**Task type:** Binary video classification (real vs. deepfake)
**Discussion:** [https://www.kaggle.com/c/deepfake-detection-challenge/discussion/140236](https://www.kaggle.com/c/deepfake-detection-challenge/discussion/140236)

**Approach:** The Deep Diggers team used WS-DAN (Weakly Supervised Data Augmentation Network) with two complementary backbones — Xception (pretrained on deepfake data) and EfficientNet-B3 (pretrained on ImageNet). RetinaFace-ResNet50 detected and aligned faces. The dramatic public-to-private rank drop (2nd to 30th) illustrates severe overfitting to the public test distribution; private deepfakes used higher-quality generation methods the model had not generalized to.

**Key Techniques:**
1. **WS-DAN (Weakly Supervised Data Augmentation Network)** — Uses attention maps to crop discriminative regions and applies feature-level augmentation. Particularly strong at localizing subtle face manipulation artifacts that naive CNN classifiers miss.
2. **RetinaFace-ResNet50 for Aligned Face Extraction** — Higher-quality face alignment compared to MTCNN; produces landmark-aligned faces that reduce pose variation and help the classifier focus on manipulation boundaries.
3. **Dual-Backbone Ensemble** — Combined WS-DAN with Xception backbone and WS-DAN with EfficientNet-B3 backbone to capture different artifact types from two complementary architectures.
4. **Frame-to-Image Dataset Pipeline** — Extracted faces from videos at training time and stored as images, allowing standard image classification training loops and data augmentation without video-specific infrastructure.
5. **~1 Week Multi-GPU Training** — Six GPU training for the primary WS-DAN model; computational scale was critical to fit the large face extraction dataset.

**How to Reuse:**
- WS-DAN / attention-guided cropping is valuable whenever fine-grained discrimination is needed (fine-grained visual recognition, medical imaging, forgery detection) — the model learns where to look.
- Public-to-private leaderboard divergence in video competitions is common when test distributions differ; rely on CV and diverse validation splits rather than optimizing for the public LB.
- For video deepfake-style tasks, invest in high-quality face alignment (RetinaFace > MTCNN in most benchmarks) as input quality directly bounds classifier performance.

## 5. Jane Street Market Prediction (2021) — 2nd Place (Survey Post)

**Task type:** Financial time-series prediction (trade profitability)
**Discussion:** [https://www.kaggle.com/c/jane-street-market-prediction/discussion/199189](https://www.kaggle.com/c/jane-street-market-prediction/discussion/199189)

**Approach:** This discussion post surveyed winning approaches from prior market prediction Kaggle challenges (including Two Sigma, G-Research) rather than presenting a single team's method. The common threads across winning market prediction solutions were: neutralization of market-wide factors, careful time-aware validation, neural network ensembles with feature neutralization, and avoiding LB probing on financial data.

**Key Techniques:**
1. **Feature Neutralization** — Regressing out market-factor exposures from features (or predictions) before training/inference to ensure the model captures idiosyncratic alpha rather than market beta. Critical in financial competitions where naive models learn the overall market trend.
2. **Time-Series Purged CV** — Using embargo-based purged K-fold (no data from adjacent time periods in train/val splits) to avoid temporal leakage. Standard k-fold dramatically overstates CV performance in financial time-series.
3. **Neural Network Ensembles with Dropout at Inference** — Multiple NN architectures (MLP, LSTM) with dropout kept on at inference time (Monte Carlo dropout) to produce uncertainty-aware predictions and naturally ensemble diverse stochastic forward passes.
4. **Regime-Aware Modeling** — Detecting market regime shifts (volatility regimes, trend/mean-reversion regimes) and training separate models or using regime as a feature, since financial signal is highly non-stationary.
5. **Prediction Clipping and Utility Optimization** — Directly optimizing a utility function (action-weighted returns) rather than MSE, since the competition metric rewarded consistent directional accuracy rather than absolute prediction accuracy.

**How to Reuse:**
- In any financial/time-series competition, build purged CV with embargo before doing any modeling — without it you're flying blind on generalization.
- Feature neutralization (projecting out known factors) is the single most reliable technique to improve out-of-sample performance in factor-based financial prediction.
- Optimize directly for the competition metric using a custom loss or post-hoc prediction transformation rather than defaulting to MSE/cross-entropy.

## 6. PetFinder.my Pawpularity Contest (2021) — 1st Place

**Task type:** Regression (predict photo engagement/popularity score 0-100)
**Discussion:** [https://www.kaggle.com/c/petfinder-pawpularity-score/discussion/300938](https://www.kaggle.com/c/petfinder-pawpularity-score/discussion/300938)

**Approach:** NVIDIA's Giba (Gilberto Titericz) won by concentrating embeddings from Swin Transformers and CNN backbones and using RAPIDS GPU-accelerated SVR as the regression head — a hybrid deep learning + classical ML approach. Rather than fine-tuning a neural regression head end-to-end, he extracted rich feature embeddings and handed them to a well-regularized SVM regressor.

**Key Techniques:**
1. **Transformer + CNN Embedding Extraction** — Fine-tuned Swin Transformer and EfficientNet/ResNet-type backbones on the pet images, then extracted penultimate-layer feature vectors. Concatenating embeddings from multiple architectures gave a rich, diverse representation.
2. **RAPIDS SVR as Regression Head** — Instead of a neural regression head, fed concatenated embeddings into RAPIDS (GPU-accelerated) Support Vector Regression. SVR's margin-based loss generalizes better than MSE on small datasets with noisy labels (popularity is inherently subjective).
3. **Two-Stage Fine-tuning** — First added an MLP regression head and fine-tuned backbone + head jointly; then froze the backbone and extracted features for SVR training. This bootstrapped better features before the SVR step.
4. **Label Normalization to [0,1]** — Normalized target from 0-100 to 0-1, applied StratifiedKFold by binned labels to ensure fold balance, and used light augmentation + TTA at inference.
5. **Multi-Model Diversity** — Ensembled SVR predictions from multiple backbone architectures (Swin variants, EfficientNets), each contributing different visual features to the final blend.

**How to Reuse:**
- For small/medium image regression tasks with noisy labels, replace the neural regression head with SVR or ridge regression on extracted embeddings — often more robust than end-to-end fine-tuning.
- RAPIDS SVR allows running classical ML at scale on GPU; use it when you need fast hyperparameter search over large embedding matrices.
- Binned stratified k-fold is the correct CV strategy for regression tasks — ensures each fold has representative label distribution, especially important with skewed targets.

## 7. TalkingData AdTracking Fraud Detection (2018) — 3rd Place (NN-based)

**Task type:** Binary classification (mobile ad click fraud)
**Discussion:** [https://www.kaggle.com/c/talkingdata-adtracking-fraud-detection/discussion/56262](https://www.kaggle.com/c/talkingdata-adtracking-fraud-detection/discussion/56262)

**Approach:** This 3rd-place solution (by a Kaggle Grandmaster) was primarily neural-network based rather than GBM, which was unusual for tabular fraud data. The key insight was that proper aggregation features (count, unique counts of app/device/channel combinations within time windows) could be expressed as low-dimensional embeddings in a neural net, avoiding the memory issues of one-hot encoding a 180M row dataset.

**Key Techniques:**
1. **NN on Aggregate Features** — Rather than feeding raw categorical IDs, computed ~20 aggregate-level features (click counts per ip/app/device/os/channel combinations within time windows) and fed them into a deep MLP. This compressed the feature space while retaining the most predictive signals.
2. **Time-Window Click Counts** — Computed rolling count features: how many times an ip/app/device combination clicked in the next 1/6/24 hours. These "next-N-clicks" features were the strongest individual predictors, capturing the burstiness pattern of bot traffic.
3. **Controlled Overfitting Prevention** — Used aggressive dropout and early stopping tuned specifically to the CV-to-LB relationship, since the huge class imbalance (fraud rate ~0.25%) made models prone to memorizing the majority class.
4. **Memory-Efficient Data Pipeline** — With 180M training rows, used chunked data loading and careful dtypes (uint8, uint16 for counts) to keep the dataset in RAM, enabling multiple training epochs.
5. **Embedding Layers for Categoricals** — Used entity embedding layers for ip/app/os/device/channel rather than frequency encoding, letting the NN learn dense representations of each category's click behavior.

**How to Reuse:**
- For large-scale click/event fraud datasets, time-window count features (next-N and previous-N clicks per entity combination) are almost always the strongest features — compute them before any modeling.
- NNs with embedding layers can outperform GBMs on high-cardinality categorical fraud data when feature space is too large for one-hot encoding; the embeddings capture co-occurrence structure.
- Aggregate-level features (~20 well-chosen ones) can match or beat hundreds of raw features in click fraud, keeping models lean and inference fast.

## 8. TalkingData AdTracking Fraud Detection (2018) — 6th Place

**Task type:** Binary classification (mobile ad click fraud)
**Discussion:** [https://www.kaggle.com/c/talkingdata-adtracking-fraud-detection/discussion/56283](https://www.kaggle.com/c/talkingdata-adtracking-fraud-detection/discussion/56283)

**Approach:** The 6th place solution used a LightGBM-centric approach with extensive click-stream feature engineering. The solution focused on constructing a large set of count, unique-count, and time-delta features over various entity combinations (ip, app, device, os, channel), then pruning aggressively with feature importance before blending LGBM models trained on different feature subsets.

**Key Techniques:**
1. **Multi-Entity Click Count Features** — Created counts and unique counts of apps/devices/channels per IP within multiple time windows (previous and next hours), covering 50+ feature combinations. Each combination captures a different facet of bot behavior.
2. **Time-Delta Features** — Computed time differences between consecutive clicks from the same ip/device/app combination. Very short or very regular inter-click intervals strongly signal automated (bot) traffic.
3. **LightGBM with Large num_leaves** — Used high-capacity LGBM (num_leaves=511, deep trees) enabled by the massive dataset size (~180M rows), with aggressive feature sampling and bagging to prevent overfitting.
4. **Feature Importance-Based Pruning** — Trained initial models with all ~200 features, then pruned to the top 80-100 by split importance before final training, reducing noise and inference time.
5. **Multi-Seed Ensemble** — Trained multiple LGBM models with different random seeds and data sampling rates, then averaged predictions. This reduced variance from the large but imbalanced dataset.

**How to Reuse:**
- In click-fraud and event-stream tasks, time-delta between consecutive events per entity is one of the most powerful fraud signals — always compute it.
- Feature importance pruning after an initial broad feature generation phase is a reliable workflow: generate 200+ features, prune to top 80, retrain.
- Large num_leaves LGBM (>200) is appropriate for massive tabular datasets (>50M rows) where you have enough data to support high-variance trees.

## 9. Global Wheat Detection (2020) — 1st Place

**Task type:** Object detection (wheat head detection in field images)
**Discussion:** [https://www.kaggle.com/c/global-wheat-detection/discussion/172418](https://www.kaggle.com/c/global-wheat-detection/discussion/172418)

**Approach:** The winning team ensembled EfficientDet variants (D5, D6, D7) with Faster RCNN FPN (ResNet-152 backbone) using Weighted Box Fusion. The solution incorporated multi-round pseudo-labeling on the test set and used external wheat datasets for pretraining warm-up, enabling a richer representation of wheat head morphology before fine-tuning on competition data.

**Key Techniques:**
1. **Multi-Model Object Detection Ensemble** — Combined EfficientDet-D5/D6/D7 (at 512–1024px) and Faster RCNN FPN (at 1024px) using Weighted Box Fusion (WBF), which merges overlapping boxes by weighted averaging coordinates rather than hard NMS suppression, retaining more detections from lower-confidence models.
2. **Iterative Pseudo-Labeling** — Round 1: trained on competition + external datasets, generated predictions on the hidden test set, used as pseudo-labels for Round 2 training. Repeated for 2 rounds, each time including pseudo-labeled test images in training, steadily improving detection coverage on unseen test distributions.
3. **External Dataset Warm-Up** — Pre-trained on the SPIKE wheat dataset and GWHD 2017 data before fine-tuning on the competition dataset. Warm-up on diverse wheat varieties improved backbone feature quality for the varied field conditions in the test set.
4. **Mosaic + MixUp Augmentation** — Applied custom Mosaic augmentation (combining 4 images into one) and MixUp to create diverse training scenes, significantly improving detection of densely packed and partially occluded wheat heads.
5. **Multi-Scale TTA** — At inference, applied horizontal flip, vertical flip, and 90° rotation augmentations, aggregating predictions across 8 TTA variants via WBF for more stable bounding box coordinates.

**How to Reuse:**
- Use WBF instead of NMS when ensembling object detection models — WBF consistently outperforms NMS in multi-model ensembles because it averages rather than discards boxes.
- Iterative pseudo-labeling on the test set (2-3 rounds) is a reliable technique in detection competitions when the test distribution differs from training — it acts as domain adaptation.
- Always warm-up detection backbones on domain-adjacent external datasets before fine-tuning on the target competition dataset, especially when competition training sets are small (<5k images).

## 10. SIIM-ISIC Melanoma Classification (2020) — 2nd Place

**Task type:** Binary classification (melanoma vs. benign skin lesion)
**Discussion:** [https://www.kaggle.com/c/siim-isic-melanoma-classification/discussion/175324](https://www.kaggle.com/c/siim-isic-melanoma-classification/discussion/175324)

**Approach:** Ian Pan's solution used EfficientNet-B6 (tf_efficientnet_b6_ns — Noisy Student pretrained) as the backbone. The key innovation was cross-year data transfer: training first on ISIC 2019 data, generating nevus labels for 2020 data using those predictions, then fine-tuning on 2020 with pseudolabeling on the unlabeled test set. Infrastructure: 4× NVIDIA Quadro RTX 6000 24GB GPUs.

**Key Techniques:**
1. **Cross-Year Transfer Learning** — Pretrained on ISIC 2019 labels, then used that model to assign nevus class labels to ISIC 2020 data (which lacked fine-grained labels). This enriched the target dataset with auxiliary class information beyond the binary melanoma label.
2. **EfficientNet-B6 Noisy Student (NS) Backbone** — The _ns variants (trained with Noisy Student self-distillation on unlabeled ImageNet) provided significantly better features than standard ImageNet pretrained weights for medical imaging, gaining 2-3% AUC on this task.
3. **Pseudolabeling on Test Data** — After initial training, generated predictions on the unlabeled test set and included high-confidence predictions as training data for subsequent rounds, effectively using all available images.
4. **K-Fold CV with Patient-Level Stratification** — Stratified folds by patient ID to prevent the same patient's multiple images from appearing in both train and validation, which would inflate CV metrics.
5. **Multi-Scale Training and TTA** — Trained at multiple resolutions; at inference applied standard TTA (horizontal flip, transpose, rotations) and averaged logits before sigmoid.

**How to Reuse:**
- When a competition releases data across multiple years/editions, always check if prior year data can be used for pretraining — cross-edition transfer is often the single biggest LB jump.
- Use patient/group-level stratification in CV for medical imaging — image-level stratification will leak if the same patient has multiple images, causing optimistic CV.
- EfficientNet Noisy Student variants (_ns) consistently outperform standard pretrained weights on fine-grained biological/medical image classification tasks.

## 11. SIIM-FISABIO-RSNA COVID-19 Detection (2021) — 1st Place (Discussion Recap)

**Task type:** Multi-label classification + object detection (COVID-19 on chest X-rays)
**Discussion:** [https://www.kaggle.com/c/siim-covid19-detection/discussion/239891](https://www.kaggle.com/c/siim-covid19-detection/discussion/239891)

**Approach:** The winning solution (dungnb1333) combined a classification pipeline and a detection pipeline in a multi-stage training scheme. Classification used ensemble of encoder-decoder models (SeR152-Unet, EfficientNet-B5/B6/B7 with DeepLab/Linknet/Unet++) for 4-class study-level prediction. Detection used YOLOv5x6, EfficientDet-D7, and Faster RCNN for opacity localization. A custom lung localizer reduced background noise. Multi-round pseudo-labeling was applied to both stages.

**Key Techniques:**
1. **Two-Task Pipeline: Classification + Detection** — Separate models for study-level classification (4 classes: negative, typical, indeterminate, atypical) and image-level opacity bounding box detection. Final submission blended both outputs, with classification label constraining which detections were reported.
2. **Lung Localizer Preprocessing** — Trained a YOLOv5-based lung detector (0.9246 mAP@0.5:0.95) to crop the lung region before classification/detection. Removing non-lung background significantly improved both tasks by reducing confounding features.
3. **Multi-Stage External Data Pretraining** — Leveraged CheXpert, ChestX-ray14, RSNA Pneumonia datasets for backbone pretraining before SIIM fine-tuning. Multi-stage: external pretraining → RSNA pneumonia fine-tune → SIIM fine-tune → pseudo-label → retrain.
4. **Iterative Pseudo-Labeling** — Selected top-2 highest-confidence detection boxes as pseudo-labels for each test image, progressively incorporating test data into training over multiple rounds.
5. **8-Variant TTA** — Applied original, center-crop 80%, H-flip, V-flip, and 4 flip combinations at inference, averaging across all 8 variants for both classification and detection outputs.

**How to Reuse:**
- For medical imaging competitions, always pretrain on the largest related public dataset (CheXpert, ImageNet-21k) before fine-tuning — domain pretraining gives far better features than generic ImageNet.
- A dedicated organ/region localizer (lung, breast, brain) as a preprocessing step almost always improves downstream classification and detection by focusing models on relevant anatomy.
- Two-task decoupled pipelines (classify first, detect within predicted positive class) consistently beat single-model approaches in medical detection competitions.

## 12. PetFinder.my Pawpularity Contest (2021) — Baseline / Early Top Solution (LB 17.91)

**Task type:** Regression (photo engagement score prediction)
**Discussion:** [https://www.kaggle.com/c/petfinder-pawpularity-score/discussion/275094](https://www.kaggle.com/c/petfinder-pawpularity-score/discussion/275094)

**Approach:** This highly upvoted baseline/early solution established the standard Swin Transformer approach that most top finishers built on. It used a Swin Transformer backbone with an MLP regression head, normalized labels to [0,1], applied StratifiedKFold by binned labels, removed duplicate images, and used fastai-style training parameters. The LB 17.91 score served as the community's reference point for the competition.

**Key Techniques:**
1. **Swin Transformer for Image Regression** — Applied Swin Transformer (a hierarchical Vision Transformer with shifted-window attention) as the feature extractor for pet photos, outperforming standard CNNs due to its ability to model long-range spatial relationships in pet cuteness cues.
2. **Label Normalization + Stratified Binning** — Normalized pawpularity scores from [0,100] to [0,1] for stable training, then binned labels into ~10 bins for StratifiedKFold to ensure each fold had a representative score distribution.
3. **Duplicate Image Removal** — Detected and removed exact duplicate images from training set (pets re-listed multiple times with different popularity scores), which would add noise to regression targets.
4. **Light Augmentation + TTA** — Used conservative augmentation (random flip, slight color jitter) to avoid corrupting fine-grained cuteness signals, with TTA (horizontal flip) at inference averaging 2 predictions per image.
5. **fastai Hyperparameter Conventions** — Adopted learning rate finder, one-cycle LR schedule, and mixed precision training from fastai, which provided well-calibrated defaults for fine-tuning pretrained Vision Transformers.

**How to Reuse:**
- For image regression with subjective labels (aesthetics, popularity, quality scores), Swin Transformer backbones consistently outperform CNNs — start with Swin-B or Swin-L pretrained on ImageNet-21k.
- Always check for and remove duplicate images in regression competitions — duplicate images with varying targets add irreducible noise that hurts CV and generalization.
- Binned StratifiedKFold is the standard CV setup for regression tasks; bin into 5-10 equal-width or equal-frequency bins before stratification.

## 13. Kaggle LLM Science Exam (2023) — 1st Place

**Task type:** Multiple-choice question answering (5-way classification, science questions)
**Discussion:** [https://www.kaggle.com/c/kaggle-llm-science-exam/discussion/446422](https://www.kaggle.com/c/kaggle-llm-science-exam/discussion/446422)

**Approach:** Team H2O LLM Studio fine-tuned an ensemble of five 7B-parameter LLMs and one 13B-parameter LLM using LoRA, combined with Retrieval-Augmented Generation (RAG) from a locally chunked Wikipedia corpus. Each model acted as a binary scorer: given (retrieved context + question + one answer choice), predict whether it's correct. Scores across all 5 choices were ranked to produce the final answer.

**Key Techniques:**
1. **LoRA Fine-tuning on All Linear Layers** — Applied LoRA (Low-Rank Adaptation) to all linear layers of 7B–13B parameter LLMs rather than just attention layers, enabling parameter-efficient fine-tuning on limited hardware while adapting the full model's representational capacity.
2. **Binary Classification Framing** — Reformulated 5-way multiple choice as binary classification: each (context, question, answer_i) triplet gets an independent "is this correct?" score. Running inference 5 times and ranking scores avoids softmax competition between answer choices.
3. **RAG from Local Wikipedia** — Chunked a local copy of Wikipedia and built a vector index; retrieved top-K relevant passages as context for each question. Different ensemble members used different embedding models and different numbers of retrieved chunks, adding diversity.
4. **Past-Key-Value Caching** — Cached the context + question as `past_key_values` and only ran the answer choice through the new forward pass, dramatically reducing inference time for the 5-answer-per-question scoring loop.
5. **Diverse 6-Model Ensemble** — Five 7B models and one 13B model, each with different configurations (data source: GPT-3.5 generated 6.5k samples, embedding model choice, chunk count). Ensemble by averaging binary classification logits across all models.

**How to Reuse:**
- For any multiple-choice LLM task, framing as binary classification (pairwise scoring) rather than 5-way softmax consistently improves accuracy by avoiding competition between similar-sounding distractors.
- RAG with diverse retrieval configurations (different embedding models, chunk sizes) is a low-cost way to create model diversity for ensembling in knowledge-intensive tasks.
- KV-cache sharing across answer choices is a critical inference optimization for LLM-based scoring; always cache the shared prefix (question + context) and only vary the answer suffix.

## 14. Mechanisms of Action (MoA) Prediction (2020) — 2nd Place

**Task type:** Multi-label classification (predict drug mechanism of action from gene/cell data)
**Discussion:** [https://www.kaggle.com/c/lish-moa/discussion/202256](https://www.kaggle.com/c/lish-moa/discussion/202256)

**Approach:** A three-model ensemble of 1D-CNN, TabNet, and DNN — each trained on gene expression + cell viability features — with weighted average blending. The 1D-CNN was the star performer at 0.01601 (private LB) as a single model; the ensemble with TabNet and DNN improved only marginally to 0.01599.
